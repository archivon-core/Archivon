# Security Policy

Archivon Core is a pre-production prototype. Please do not use it to protect sensitive production data without independent security review and deployment hardening.

## Reporting a Vulnerability

If GitHub private vulnerability reporting is enabled for this repository, please use it for security reports.

Do not open public issues containing exploit details, secrets, credentials, private infrastructure information, screenshots with sensitive data, or proof-of-concept payloads against real systems.

If private reporting is not yet enabled, wait until an anonymous project contact or GitHub Security Advisory workflow is published for this repository.

## Scope

Security reports are welcome for:

- authentication and session handling;
- Proof-of-Work access policy enforcement;
- WebDAV access controls;
- audit integrity;
- migration safety;
- deployment template hardening;
- secret handling.

Out of scope for the first public prototype:

- attacks against private deployments not owned by the reporter;
- social engineering;
- denial-of-service testing against public infrastructure;
- reports requiring exposure of personal identity, private infrastructure, logs, or credentials.
